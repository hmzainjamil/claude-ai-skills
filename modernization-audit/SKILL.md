# Modernization Audit — Legacy System Analysis
## Triggers: modernization audit, legacy system, old system, tech audit, stack audit, modernize

## AUDIT FRAMEWORK
1. Current stack inventory (languages, frameworks, services)
2. Pain points (slow deploys, frequent bugs, scaling limits)
3. Bottlenecks (manual processes, single points of failure)
4. Modernization options (rewrite vs refactor vs replace)
5. ROI estimate (dev hours saved, reliability improvement)
6. Roadmap (30/60/90 day phases)

## QUICK WINS (no-rewrite)
- Add CI/CD pipeline (GitHub Actions)
- Containerize with Docker
- Add monitoring (Grafana/Datadog)
- Extract microservices for bottleneck modules

## FULL MODERNIZATION
Weeks 1-4: API layer + documentation
Weeks 5-8: Frontend modernization (React/Next.js)
Weeks 9-12: Data layer + caching
